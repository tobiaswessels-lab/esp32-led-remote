How it works — step by step

1
User opens the web app — Browser loads index.html from GitHub Pages. This is a static file with HTML, CSS, and JavaScript. No server-side code.
2
Web app connects to MQTT broker — JavaScript uses mqtt.js library to open a WebSocket connection (wss:// port 8884) to HiveMQ Cloud. Authenticated with username + password.
3
User taps a button — The web app publishes a command (e.g. text:HELLO) to MQTT topic esp32/led/cmd.
4
HiveMQ relays to ESP32 — The broker forwards the message to all subscribers of esp32/led/cmd. The ESP32 is subscribed via TLS on port 8883.
5
ESP32 executes the command — The handleCommand() function processes the message (same handler as the local WebSocket) and updates the LED matrix via SPI.
6
ESP32 sends status back — Publishes display status and live pixel data (32 bytes as hex) to esp32/led/status. HiveMQ relays it to all connected browsers.
7
Browser renders live preview — The web app decodes the pixel data and draws a red LED matrix on a canvas, matching the physical display including brightness level.
✓
No port forwarding needed — Both the browser and ESP32 connect outbound to HiveMQ. The home router doesn't need any configuration. All connections are encrypted (TLS/WSS).
